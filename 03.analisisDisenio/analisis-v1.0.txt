	* pantalla: reservaLugar
		en esta pantalla el usuario podra aceptar o revocar las reservas de lugares que le realicen. Tambien podra eliminar reservar
		si el cliente no cumple con el horario
		
		+ se visualizara un listado de las reservas confirmadas. con las siguientes columnas:
			- id de la reserva
			- id de la patente - nombre en caso de ser bicicleta
			- tipo vehiculo

		+ se visualizara un listado de las reservas pendientes
			- id de la reserva
			- id de la patente - nombre en caso de ser bicicleta
			- tipo vehiculo
			- descripcion vehiculo
			- fecha y hora de entrada
			- tolerancia solicitada(minutos)
			- fecha y hora de salida
			- estado (pendiente, por finalizar espera)
			- acciones (aceptar reserva, rechazar reserva)
		+ al presionar el boton de la accion revocar saldra un popUp o ventana modal que solicitara un motivo por el cual se revoca
		la reserva. Esta ventana o popUp tendra un boton aceptar y otro cancelar. Al presionar el cancelar la ventana o popUp se cerrara
		y no se realizara ninguna accion. Al presionar el boton aceptar, se validara que haya algun motivo de revocacion y esto eliminara
		la reserva (la pasa a estado Baja) de la base de datos y de la lista de reservas confirmadas. Posterior a esto se enviara una
		notificacion al server para que le envie al celular que realizo la reserva la notificacion de que la reserva fue revocada.
		+ al presionar el boton de la accion aceptar reserva, se modificara en la base de datos el estado de la reserva a Confirmada,
		en la pantalla se eliminara el registro de la lista de reservas pendientes y se agregara a la lista de reservas confirmadas. Posterior
		a esto se enviara una notificacion al server para que le informe al celular que realizo la reserva que esta fue confirmada
		+ al presionar el boton de la accion rechazar reserva saldra un popUp o ventana modal que solicitara el motivo del rechazo de la
		reserva, esta vantana o popUp tendra dos botones, uno guardar y otro cancelar. Al presionar cancelar la ventana o popUp se cerrara
		y no se realizara ninguna otra accion. En caso de que se presiono guardar, se validara que haya un motivo por el rechazo de la reserva
		y se cambiara el estado a Rechazada. Posteriormente se eliminara la reserva del listado de reservas pendientes y se enviara una 
		notificacion al server para que le envie un mensaje al celular que realizo la reserva indicando que esta fue rechazada
		+ aquellas reservas que sobrepasen un periodo de tiempo definido seran rechazadas automaticamente por el server indicando
		como motivo de rechazo: falta de confirmacion del estacionamiento
